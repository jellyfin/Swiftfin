- [ ] **DISCUSS: UIKit-drawn shelf cells — the snappiness ceiling (possible rearchitecture, owner may pursue)** — verified via swift-xcode-expert (2026-06-25, on-device HUD data: 60fps settled, ~20fps/50–117ms scrolling, single 1285ms·76f / 423ms·24f freezes on surface-enter / decade-select). Root cause is the per-cell `UIHostingController` mint: `HostingCollectionViewCell.setup` builds a fresh `UIHostingController` on **every** `cellForItemAt` and discards it in `prepareForReuse`, so recycling never amortizes the SwiftUI-graph build — the freeze scales with how many cells are built in one synchronous update, NOT with the collection topology. This cost is **identical in stock Swiftfin** (stock tvOS `HomeView` is the same `ScrollView{VStack{ForEach{PosterHStack}}}`; "see all" = one flat paged `CollectionVGrid`). The LePips package has **no** vertical-of-horizontal recycling primitive and `CollectionVList` is iOS-only, so the mooted "single recycling collection" rewrite (PERF_SHELVES #5) neither exists off-the-shelf nor removes the per-cell cost — **strike #5 / correct the PERF_SHELVES claim that stock uses one collection**. What Netflix/Apple-TV-app do that Bruno can't today: (a) truly-recycled lightweight UIKit cells (UIImageView+label, image swapped on reuse — no per-cell controller mint), (b) one `UICollectionViewCompositionalLayout` with `.orthogonalScrollingBehavior(.continuous)` per row = one reuse pool for the whole page, (c) zero layout/decode/effect work during the focus move (art pre-decoded off-main to exact size, prefetched ≥1 screen ahead, baked shadows). Matching that = drop the poster cell to a recycled UIKit cell under a compositional layout: **multi-week, highest regression risk** (focus engine, Show-all trailing card, gradient category row, zoom transitions, every INV-1..9). ## Spine vs tail (the product contract, condensed)

Home is a **stable spine** (shelves 2–10/11: Continue Watching → Up Next → New Releases → Director →
Genre → Series → Studio → Eras → Browse-by-Director → Collections) plus a **dynamic tail** of 5–6 seeded
"explore" shelves (+2 per scroll page, cap ~18). `seed = dayStamp()`, **Shuffle** re-rolls. `buildHome(seed)`
is pure given `(seed, librarySnapshot)`. Full taxonomy + generators: `PRODUCT_SPEC.md` §3–4.

## Guardrails (do not regress)

- **Additive, tvOS-only.** Bruno UI lives in `Swiftfin tvOS/Views/BrunoHomeView/` and `Shared/Objects/Bruno/`.
  iOS stays stock Swiftfin + rebrand. Don't touch player/nav/detail engine source.
- New files in the file-system-synchronized tvOS group → **no `.pbxproj` edits**. Non-Bruno edits stay
  DEBUG-gated and inert-by-default (see `SwiftfinApp.swift`).
- Never hardcode BoxSet/library IDs — discover groups dynamically (favorited BoxSets of BoxSets).
- **Before UX-polishing Home/browse shelves, read `docs/BRUNO_PERF_INVARIANTS.md`** (INV-1..9). The
  Home perf rests on non-obvious invariants (fixed row height, stable shelf ids, prefetch width ==
  cell width, seed-keyed/source-restricted cache); `// INV-n` anchors mark each site. Restyle freely —
  just keep those nine intact.
  - **Owner-override exception:** `BrunoStudiosGridView` *intentionally* breaks **INV-6** (scroll-coupled
